\chapter{PROJECT IMPLEMENTATION}

\section{Project Title}
\textbf{Event-Driven Serverless EC2 Cost Optimization System on AWS}

\section{Overview}
This project implements an automated, serverless solution on AWS to optimize cloud costs by identifying and stopping idle EC2 instances. It leverages AWS native services to ensure scalability, efficiency, and minimal operational overhead.

\section{Problem Statement}
In cloud environments, EC2 instances often remain running even when not actively used, leading to unnecessary costs. Manually monitoring and managing these instances is inefficient and error-prone.

\section{Objective}
To design and implement an automated system that:
\begin{itemize}[leftmargin=1.5cm]
    \item Detects idle EC2 instances
    \item Stops non-essential instances based on usage patterns
    \item Reduces cloud infrastructure costs
    \item Operates without manual intervention
\end{itemize}

\section{Architecture \& Services Used}
\begin{itemize}[leftmargin=1.5cm]
    \item \textbf{AWS EventBridge} -- Schedules periodic triggers
    \item \textbf{AWS Lambda} -- Executes logic for monitoring and control
    \item \textbf{Amazon EC2} -- Compute resources being monitored
    \item \textbf{Amazon CloudWatch} -- Provides CPU utilization metrics
    \item \textbf{AWS IAM} -- Manages roles and permissions
    \item \textbf{AWS Tags} -- Filters non-production instances
\end{itemize}

\section{System Workflow}
\begin{enumerate}[leftmargin=1.5cm]
    \item EventBridge triggers the Lambda function at defined intervals
    \item Lambda fetches the list of EC2 instances
    \item Filters instances based on tags (e.g., non-production)
    \item Retrieves CPU utilization metrics from CloudWatch
    \item Compares usage against a predefined threshold
    \item Stops instances identified as idle
    \item Logs all actions for monitoring and auditing
\end{enumerate}

\section{Key Features}
\begin{itemize}[leftmargin=1.5cm]
    \item \textbf{Fully Serverless} -- No infrastructure management required
    \item \textbf{Automated Scheduling} -- Runs periodically using EventBridge
    \item \textbf{Cost Optimization} -- Reduces unnecessary cloud spending
    \item \textbf{Tag-Based Filtering} -- Ensures production safety
    \item \textbf{Scalable Design} -- Works across multiple instances
    \item \textbf{Monitoring \& Logging} -- Tracks actions using CloudWatch Logs
\end{itemize}

\section{Idle Detection Logic}
Idle instances are determined based on CPU utilization thresholds:
\begin{itemize}[leftmargin=1.5cm]
    \item If CPU usage remains below a defined threshold (e.g., 5--10\%) over a specific duration
    \item The instance is classified as idle
    \item The system automatically stops the instance
\end{itemize}

\section{Challenges \& Solutions}

\noindent\textbf{Challenge:} Defining what qualifies as an ``idle'' instance\\
\textbf{Solution:}
\begin{itemize}[leftmargin=1.5cm]
    \item Implemented CPU utilization thresholds
    \item Used CloudWatch metrics over a time window instead of instant values
\end{itemize}

\noindent\textbf{Challenge:} Avoid affecting critical systems\\
\textbf{Solution:}
\begin{itemize}[leftmargin=1.5cm]
    \item Applied tag-based filtering to exclude production resources
\end{itemize}

\section{Results \& Impact}
\begin{itemize}[leftmargin=1.5cm]
    \item Reduced unnecessary EC2 runtime costs
    \item Eliminated manual monitoring effort
    \item Improved operational efficiency
    \item Demonstrated effective use of serverless architecture
\end{itemize}

\section{Future Enhancements}
\begin{itemize}[leftmargin=1.5cm]
    \item Add auto-start functionality based on schedules
    \item Integrate notification system (SNS/Email alerts)
    \item Use machine learning for smarter idle detection
    \item Extend support to other AWS resources (RDS, ECS)
\end{itemize}

\section{Conclusion}
This project demonstrates how an event-driven, serverless architecture can effectively automate infrastructure management tasks. By combining AWS services like EventBridge, Lambda, and CloudWatch, the solution achieves cost optimization while maintaining scalability and reliability.

modern intelligent systems.

This internship also helped bridge the gap between theoretical knowledge and real-world 
technology applications. Concepts learned in areas such as networking, databases, and 
software systems could be related to practical cloud services provided by AWS. By 
studying these technologies together, I developed a clearer understanding of how cloud

The methodology followed during the internship was based on a structured learning 
approach through the Amazon Web Services Educate training modules. Each module
